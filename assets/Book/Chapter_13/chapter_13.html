1. The Last Declaration Will Win

When two CSS rules have the same specificity, the one declared last will override the others.

<style>
  .priority {
    color: red;
  }
  .priority {
    color: blue;
  }
</style>
<p class="priority">This will be blue</p>

This will be blue

2. The More Specific Declaration Will Win

ID selectors are more specific than class selectors, so they take precedence.

<style>
  .specific {
    color: green;
  }
  #specific {
    color: orange;
  }
</style>
<p id="specific" class="specific">This will be orange</p>

This will be orange

3. Inline Style is Preferred to Both Internal and External Style

Inline styles override all rules defined in internal or external CSS files.

<p style="color: purple;">This will be purple</p>

This will be purple

4. "!important" Declaration is the Strongest Rule

The !important rule overrides all other rules, including inline styles.

<style>
  .important-output {
    color: blue !important;
  }
  .important-output {
    color: red !important;
  }
</style>
<p class="important-output" style="color: green;">This will be red</p>

This will be red
